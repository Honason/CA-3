Testing

Frontend testing

We have used Karma combined with Jasmine to create automated testing of our frontend functions. We found that an application of this scale that tests were mostly a waste of time since we could easily test manually. The right time to construct these tests would be right before starting to code the application itself so that we could test our code against the test instead of testing the test against the code.

There were not a lot of frontend functionality, most importantly was that the calculator converted correctly and the login system handled tokens without fault.

Tests:

  • CVRAPI
  • No input in the search query
  • Correct data allocation when returned from api
  • Login
  • Logout button deleting token from browser
  • Calculator
  • Correctly convert between currencies
  • Only show 2 decimals

Manual API testing

To simplify the process and save time, we have decided to conduct the simple API testing of register functionality in Postman application. We have selected four test cases to ensure that system behaves as it should.

Register API path expects json object as a parameter in body, which contains username and password values. In first test, we wont set this object. Instead, we will do empty POST request to api register path. Server returns 400 (bad request) error, and information about the error.

In second test, we will try to send just on of the username-password pair. The server should return the same http 400 status, with message "Incomplete request".

In third test, we will pass valid object with both username and password, but with username that is already registered. Expected result is again status 400, and message "User already exist".

The fourth and last test case passes valid object with username and password, with username that doesn't exist in our database. Expected response is OK status 200, and same json response object as login path. Username and token.
